# hi, i'm alisha! 

i’m a student, builder, and a bit of a physics nerd. my desk is usually a mess of breadboards, half-finished drawings, and open textbooks ( i don't read them though)

### what i'm up to
most of my time is spent staring at a screen wondering why a dot won't move, or staring at a breadboard wondering why a led won't blink. but i love the "raw" way of doing things. if i can’t see the math or the wires, i don't feel like i'm actually learning it.

*   **software:** currently obsessed with "raw" frontend. if i can replicate a complex physics diagram with just a few divs and some trigonometry, i'm happy. 
*   **hardware:** i love the physical side of things too. there's something special about writing code that actually moves a motor or lights up a led in the real world.
*   **art & cading:** i want to get better at 3d modeling (cading) so i can design actual enclosures for my hardware. i also hand-draw my own cursors and asset. i think tech should have more personality, so i try to leave a bit of my sketchbook in every repo.

### my 2026 list
i'm using this year to stop "guessing" and start actually knowing. specifically, i'm going to:
- master the math: i want to get so comfortable with trig that i can animate things like wave interference or pendulums without looking at a tutorial every five minutes.
- stop frying components: in hardware, i want to move past "following the diagram" and actually understand how to calculate my own resistor values and design circuits that don't go up in smoke.
- hand-drawn ui: i’m working on making my digital project, i want to learn how to properly integrate my hand-drawn art so the web feels more tactile and less corporate.
